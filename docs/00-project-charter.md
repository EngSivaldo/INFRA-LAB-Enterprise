# Project Charter — INFRA-LAB Enterprise

## 1. Identificação do Projeto

**Nome do Projeto:** INFRA-LAB Enterprise

**Tipo:** Laboratório Profissional de Infraestrutura Linux, DevOps e Cloud

**Responsável:** Sivaldo Vieira de Almeida

**Objetivo:** Construir uma infraestrutura corporativa simulada utilizando tecnologias utilizadas no mercado, documentando todo o processo de planejamento, implementação, testes e troubleshooting.

---

# 2. Visão Geral

O INFRA-LAB Enterprise representa a construção de um ambiente de infraestrutura empresarial utilizando máquinas virtuais Linux.

O projeto tem como objetivo reproduzir cenários encontrados em ambientes reais de empresas, incluindo:

- servidores Linux;
- redes internas;
- serviços de infraestrutura;
- servidores web;
- segurança;
- automação;
- práticas DevOps.

Todo o desenvolvimento será documentado e versionado utilizando Git e GitHub.

---

# 3. Objetivos do Projeto

## Objetivo Geral

Criar um laboratório completo de infraestrutura que demonstre conhecimentos práticos em administração Linux, redes, servidores e DevOps.

## Objetivos Específicos

- Configurar servidores Linux.
- Implementar serviços de rede.
- Documentar arquitetura e decisões técnicas.
- Criar scripts de automação.
- Aplicar boas práticas de administração.
- Desenvolver capacidade de troubleshooting.
- Criar um portfólio técnico profissional.

---

# 6. Metodologia

O projeto seguirá uma abordagem profissional:

1. Planejamento.
2. Documentação.
3. Implementação.
4. Testes.
5. Validação.
6. Troubleshooting.
7. Versionamento.

Cada etapa deverá possuir:

- comandos utilizados;
- testes realizados;
- problemas encontrados;
- soluções aplicadas.

---

# 7. Critérios de Sucesso

O projeto será considerado concluído quando possuir:

- infraestrutura funcional;
- documentação completa;
- diagramas técnicos;
- scripts automatizados;
- histórico de evolução no GitHub;
- capacidade de reprodução do ambiente.

---

# 8. Controle de Versão

O projeto utiliza:

- Git para versionamento local.
- GitHub como repositório remoto.
- Branch principal: main.

Todos os avanços importantes deverão gerar commits organizados.
